## 1️⃣ 项目名称 / Project Name

* 中文名：**FitEasy** / **健身易**
* 英文名：**FitEasy**

  * 含义：简单易用、记录健身、快速上手

---

## 2️⃣ 目标用户 / Target Users

* 健身初学者，希望记录训练数据
* 健身爱好者，希望跟踪进展
* 希望快速查看和分享自己健身成果的人

---

## 3️⃣ 核心功能 / Core Features

| 功能模块    | 功能点     | 说明                           |
| ------- | ------- | ---------------------------- |
| 用户管理    | 注册 / 登录 | 账号密码 |
| 健身记录    | 添加训练    | 支持自定义训练项目（如卧推、深蹲、跑步）         |
|         | 编辑 / 删除 | 可修改或删除已记录的训练                 |
|         | 自动统计    | 根据重量、次数、距离、时间生成简单进度统计        |
| 数据可视化   | 图表展示    | 柱状图或折线图展示训练重量、次数、里程等趋势       |
|         | 周 / 月汇总 | 自动生成每周 / 每月训练汇总              |
| 健身计划    | 简单模板    | 提供常用训练计划模板，可快速导入             |
|         | 自定义计划   | 用户可自己创建训练计划并打卡               |
| 提醒功能    | 训练提醒    | 每日/每周定时提醒用户训练                |
| 社交 / 分享 | 生成图片    | 可生成训练成果图分享到朋友圈               |
|         | 进度分享    | 可分享训练进度（非必需，可后期加）            |

---

## 4️⃣ 页面布局 / UI Layout

**目标：经典布局 + 低开发成本**

* **底部 Tab 栏**（代替侧边栏，更适合移动端）

  1. 首页 / Dashboard：训练概览、进度统计
  2. 训练记录 / Record：列表 & 添加训练
  3. 健身计划 / Plan：训练计划模板 & 自定义计划
  4. 数据分析 / Stats：图表展示趋势
  5. 我的 / Profile：用户信息、设置、分享

* **首页布局建议**：

  * 今日训练摘要
  * 本周训练趋势折线图
  * 快速添加训练按钮

* **记录页面**：

  * 按日期列表显示训练
  * 支持快速添加、修改和删除

* **计划页面**：

  * 模板训练计划列表
  * 可创建并打卡自定义计划

* **数据页面**：

  * 柱状图显示每周重量/次数
  * 折线图显示跑步里程/时长趋势

* **设置页面**：

  * 主题切换：Light / Dark
  * 提醒设置
  * 数据导出（CSV/PDF）

---

## 5️⃣ 技术实现建议 / Tech Stack

* **前端**：ArkTS + HarmonyOS UI Kit（符合鸿蒙开发环境）
* **后端**：可选轻量 Node.js / 云函数存储记录
* **存储**：本地 SQLite 或轻量云数据库
* **图表**：ECharts / 图表控件
* **提醒**：系统本地通知 API
* **分享**：生成图片 + 系统分享 API

---

## 6️⃣ 开发难度优化策略 / Reduce Dev Complexity

* 使用系统控件 + ArkTS 标准组件，避免自定义复杂动画
* 图表只需折线/柱状图，不需要复杂交互
* 用户注册可使用匿名模式 + 第三方登录
* 初版功能先不做社交，只生成图片分享即可

---

## 7️⃣ 增强用户体验 / Optional Enhancements

* 每日打卡勋章 / 成就系统
* 训练数据导入导出（JSON/CSV）
* 训练历史导入（可后期从其他 App 导入）

## 1️⃣ AI 辅助功能定位

* 功能名称（预留入口）：

  * 中文：**智能训练助手**
  * 英文：**AI Fitness Coach**
* 目标用户：

  * 希望得到训练建议或优化训练计划的用户
* 功能类型：

  * **可点击的入口按钮/Tab**，暂时显示“敬请期待”或简单提示
  * 后续版本可实现：

    * 自动生成训练计划
    * 动作建议与纠正
    * 训练数据智能分析
    * AI 问答：如何提高力量、耐力等

**UI 提示**：

* 文案示例：

  * “Coming soon: AI Fitness Coach”

## 3️⃣ 技术实现保留点

* 前端：

  * 预留 AI 模块的路由或页面组件
  * 底层接口接口占位（比如 `/api/ai/coach`）
  * 设计数据模型字段，如 `aiRecommendation: string`，暂时为空
* 后端：

  * 先不实现复杂逻辑，只保留接口占位
  * 日后可接入 AI 模型（如本地轻量推理或调用云端大模型）

## 4️⃣ 第二版 AI 功能可拓展点

* **训练计划生成**：

  * 用户输入目标（增肌 / 减脂 / 耐力）
  * AI 推荐训练计划 + 周期
* **动作优化建议**：

  * 输入训练数据
  * AI 给出重量、次数、组数调整建议
* **智能提醒**：

  * AI 根据用户训练频率与历史表现，智能提醒训练或休息
* **训练问答**：

  * 用户提问训练问题
  * AI 即时回答
